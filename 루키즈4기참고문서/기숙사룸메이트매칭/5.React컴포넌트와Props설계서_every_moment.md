## React 컴포넌트 개요

### 컴포넌트명

everymoment

### 목적 및 역할

- 로그인/회원가입을 통해 사용자가 자유/공지 등 4가지의 게시판, 룸메이트 매칭, 룸메이트 매칭을 위한 설문조사, 룸메이트 매칭을 위한 채팅 기능 등 다양한 기능을 이용할 수 있는 커뮤니티 웹 서비스를 제공

### 기능 요구사항

- [기능 1] 사용자 인증 (로그인/로그아웃/회원가입/프로필 상세 및 수정)
- [기능 2] 게시판 기능 (목록/상세/작성/수정/삭제, 댓글 기능)
- [기능 3] 설문조사 기반 룸메이트 매칭 결과 조회 및 표시
- [기능 4] 설문 응답 및 결과 표시
- [기능 5] 룸메이트를 찾기 위한 채팅 (요청 알림, 수락/거절 결과 노출)기능 제공
- [기능 6] ProtectedRoute를 통해 인증 사용자만 접근 가능하도록 페이지 보호

## React 컴포넌트 Props 설계

### Props 인터페이스

| Props명        | 타입                              | 필수여부             | 기본값    | 설명                               |
| -------------- | --------------------------------- | -------------------- | --------- | ---------------------------------- |
| currentUser    | object                            | 선택                 | null      | 현재 로그인 사용자 정보            |
| isAdmin        | boolean                           | 선택                 | false     | 관리자 권한 여부(역할기반 UI/가드) |
| status         | 'PENDING'\|'ACCEPTED'\|'REJECTED' | 선택                 | 'PENDING' | 요청/게시글/매칭 상태 배지 표시용  |
| onSubmit       | function                          | 선택                 | -         | 폼/버튼 제출 핸들러                |
| onLogout       | function                          | 선택                 | -         | 로그아웃 실행 함수                 |
| onApprove      | function(id)                      | 선택 (관리자용)      | -         | 스왑/요청 승인                     |
| onReject       | function(id)                      | 선택 (관리자용)      | -         | 스왑/요청 거절                     |
| id (useParams) | string                            | 필수(상세/수정 화면) | -         | 라우팅 파라미터                    |

### API 연동

| API 엔드포인트           | 메서드 | 용도                    | 호출 시점             |
| ------------------------ | ------ | ----------------------- | --------------------- |
| `/api/user/login`        | POST   | 로그인                  | 로그인 버튼 클릭 시   |
| `/api/user/register`     | POST   | 회원가입                | 회원가입 제출 시      |
| `/api/user/me`           | GET    | 현재 사용자 정보        | 프로필/헤더 로드      |
| `/api/posts`             | GET    | 게시글 목록             | BoardPage 로드        |
| `/api/posts/:id`         | GET    | 게시글 상세             | BoardDetailPage 진입  |
| `/api/posts`             | POST   | 게시글 작성             | 작성 저장             |
| `/api/posts/:id`         | PUT    | 게시글 수정             | 수정 저장             |
| `/api/posts/:id`         | DELETE | 게시글 삭제             | 상세에서 삭제         |
| `/api/posts/:id/approve` | POST   | 스왑/요청 승인 (관리자) | 상세에서 승인 클릭    |
| `/api/posts/:id/reject`  | POST   | 스왑/요청 거절 (관리자) | 상세에서 거절 클릭    |
| `/api/comments`          | POST   | 댓글 등록               | 상세에서 등록         |
| `/api/comments/:id`      | DELETE | 댓글 삭제               | 상세에서 삭제         |
| `/api/match`             | GET    | 매칭/추천 결과          | MatchResultsPage 로드 |
| `/api/survey`            | POST   | 설문 응답 제출          | SurveyPage 제출       |
| `/api/survey/result`     | GET    | 설문 결과 조회          | SurveyResultPage 로드 |
| `/api/chat/rooms`        | GET    | 채팅방 목록             | ChatPage 로드         |
| `/api/chat/messages`     | GET    | 특정 방 메시지 조회     | 방 진입 시            |
| `/api/chat/messages`     | POST   | 메시지 전송             | 입력 시               |

### 성능 최적화

- [1] React.memo로 불필요한 렌더링 최소화
- [2] useMemo / useCallback으로 연산 및 핸들러 메모이제이션
- [3] 리스트 랜더링 시 key 안정화 및 페이지네이션 적용
- [4] Zustand와 로컬 상태를 구분하여 최소 범위로 관리
- [5] axiosInstance 기반 공통 요청 처리 및 응답 캐싱 고려
- [6] css 모듈 기반 스타일 적용으로 성능 및 유지보수성 향상
