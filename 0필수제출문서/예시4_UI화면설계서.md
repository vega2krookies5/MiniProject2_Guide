#### 3.1.3 UI 컴포넌트 매핑
| 화면 요소 | React 컴포넌트 | 라이브러리 컴포넌트 | Props / 비고 |
|-----------|---------------|-------------------|------|
| 검색 입력창 | `BookSearch` | `Input` (shadcn/ui) | `onSearch(keyword)` |
| 카테고리 필터 | `CategoryFilter` | `Select` (shadcn/ui) | `onSelect(category)` |
| 도서 테이블 | `BookDataGrid` | MUI DataGrid | `rows={books}`, 서버사이드 페이지네이션 |
| 재고 뱃지 | 인라인 `Badge` | `Badge` (shadcn/ui) | availableCopies > 0 → `success`, 0 → `destructive` |
| 대출 버튼 | `LoanButton` | `Button` (shadcn/ui) | 비로그인 시 disabled + 툴팁 |
| 대출 신청 모달 | `LoanDialog` | `Dialog` (shadcn/ui) | `bookId`, `bookTitle`, `onConfirm` |
| 페이지네이션 | MUI DataGrid 내장 | MUI Pagination | `page`, `pageSize`, `rowCount` |

#### 3.1.4 Zustand Store 연결
```javascript
// useBookStore 사용
const { books, loading, pageNo, totalPages, fetchBooks } = useBookStore();
const { token } = useAuthStore();

// 페이지 진입 시
useEffect(() => {
  fetchBooks({ keyword: '', category: '', page: 0 });
}, []);
```

#### 3.1.5 화면 상태 목록
| 상태 | 표시 내용 | 트리거 조건 |
|------|-----------|-------------|
| **초기 로딩** | Skeleton(뼈대 UI) 표시 | 컴포넌트 마운트 시 |
| **정상 표시** | 도서 목록 테이블 | API 성공 + 데이터 1건 이상 |
| **검색 결과 없음** | "검색 결과가 없습니다. 다른 키워드로 검색해보세요." | API 성공 + 데이터 0건 |
| **에러** | "오류가 발생했습니다. 잠시 후 다시 시도해 주세요." + 재시도 버튼 | API 호출 실패 |

#### 3.1.6 사용자 인터랙션 흐름
```
[검색어 입력]
    → Enter 또는 검색 버튼 클릭
    → fetchBooks({ keyword, category, page: 0 })
    → API: GET /api/books?keyword=클린코드&page=0
    → loading=true → 테이블 갱신 → loading=false

[카테고리 선택]
    → setCategory(value)
    → fetchBooks({ keyword, category: 'IT', page: 0 })

[대출 버튼 클릭] — 비로그인
    → 토스트: "로그인이 필요합니다"
    → navigate('/login')

[대출 버튼 클릭] — 로그인
    → setSelectedBook(book)
    → LoanDialog open=true
    → [확인 클릭] → POST /api/loans { bookId: 1 }
    → 성공: 토스트 "대출 신청이 완료되었습니다" + Dialog close
    → 실패 (재고 없음): 토스트 "대출 가능한 도서가 없습니다"
    → 실패 (한도 초과): 토스트 "최대 대출 가능 권수를 초과했습니다"
```

| `CANCELLED` | `secondary` (회색) | 취소됨 |

#### 3.3.5 사용자 인터랙션 흐름
```

    → 대출이력 테이블 표시

[상태 탭 클릭] (예: "대출중")
    → setStatusFilter('BORROWED')
    → loans를 클라이언트 사이드 필터링

[반납 버튼 클릭]
    → confirm("반납 신청하시겠습니까?")
    → [확인] → returnBook(loanId)
    → PATCH /api/loans/1/return
    → 성공: 토스트 "반납 신청이 완료되었습니다" + 목록 갱신
    → 실패: 토스트 에러 메시지
```

## 4. 네비게이션 흐름도

```
[도서 목록 /]  ←→  [도서 상세 /books/:id]
    │                     │
    │            [대출 신청 Dialog]
    │                     │ 성공
    │                     ↓
    │            [내 대출이력 /my/loans]
    │                     │
    │                [반납 신청]
    │
    ├── [로그인 /login]  ←  (비로그인 대출 시도)
    │         │ 성공
    │         └── [이전 페이지 또는 /]
    │
    └── [회원가입 /register]
              │ 성공
              └── [로그인 /login]
```
