## 1. 프로젝트 개요

### 1.1 프로젝트 목적

도서관 회원들이 언제 어디서나 도서 검색·대출·반납 신청을 온라인으로 처리할 수 있도록 하고,
관리자의 도서 재고 및 대출 현황 관리 업무를 디지털화하여 운영 효율을 높인다.

### 1.2 프로젝트 범위

**포함 범위:**
- 회원 관리 — 가입, 로그인(JWT), 프로필 수정, 탈퇴
- 도서 관리 — 등록, 수정, 삭제, 검색(제목·저자·카테고리)
- 대출/반납 관리 — 대출 신청, 관리자 승인, 반납, 연체 관리
- 카테고리 관리 — 소설 / IT / 역사 / 과학 / 기타

**제외 범위:**
- 결제 시스템 (연체료 결제는 오프라인 처리)
- 실시간 알림 (이메일 알림은 2차 개발)
- 도서 예약 시스템 (2차 개발)

### 1.3 주요 이해관계자

| 구분 | 역할 | 주요 관심사 |
|------|------|-------------|
| **도서관 회원** | 시스템 사용자 | 도서 검색, 대출·반납 신청, 대출이력 조회 |
| **도서관 관리자** | 시스템 운영자 | 도서 재고 관리, 대출 승인·반납 처리, 연체 관리 |
| **IT 담당자** | 시스템 관리자 | 서버 운영, 보안, 백업, 확장성 |

## 2. 비즈니스 도메인 분석

### 2.1 핵심 비즈니스 프로세스

#### 2.1.1 도서 대출 프로세스

```
[회원] 도서 검색 → 대출 신청
    ↓
[시스템] 재고 확인 (availableCopies > 0)
    ↓
[관리자] 대출 승인 (REQUESTED → BORROWED)
    ↓
[시스템] 대출 기간 14일 설정, 반납 예정일 저장
    ↓
[회원] 14일 이내 반납 신청
    ↓
[관리자] 반납 처리 (BORROWED → RETURNED)
```

**비즈니스 규칙:**
- 회원은 최대 5권까지 동시 대출 가능
- 대출 기간은 14일, 1회 연장 가능 (최대 7일)
- 반납일 초과 시 하루 100원의 연체료 발생
- 연체 중인 회원은 신규 대출 불가

#### 2.1.2 회원 가입 / 로그인 프로세스

```
[사용자] 이메일 + 비밀번호 입력
    ↓
[시스템] 이메일 중복 확인 → 비밀번호 BCrypt 암호화
    ↓
[시스템] 회원번호 자동 생성 (M + 9자리 숫자)
    ↓
[사용자] 로그인 → JWT 액세스 토큰(1시간) + 리프레시 토큰(7일) 발급
```

### 2.2 비즈니스 이벤트

| 이벤트 | 트리거 | 결과 |
|--------|--------|------|
| **회원 가입** | 이메일/비밀번호 입력 후 가입 버튼 | 회원 생성, 회원번호 자동 발급 |
| **로그인 성공** | 이메일/비밀번호 일치 | JWT 액세스·리프레시 토큰 발급 |
| **대출 신청** | 회원이 도서 대출 버튼 클릭 | Loan 레코드 생성 (status=REQUESTED), 재고 감소 |
| **대출 승인** | 관리자가 승인 처리 | Loan status: REQUESTED → BORROWED |
| **반납 처리** | 관리자가 반납 처리 | Loan status: BORROWED → RETURNED, 재고 증가 |
| **연체 발생** | 반납 예정일 초과 | Loan status: BORROWED → OVERDUE, 신규 대출 차단 |
| **회원 정지** | 관리자가 회원 상태 변경 | Member status: ACTIVE → SUSPENDED |

## 3. 핵심 도메인 객체

### 3.1 도메인 객체 식별 매트릭스

| 도메인 객체 | 유형 | 중요도 | 복잡도 | 연관관계 수 | 우선순위 |
|-------------|------|--------|--------|-------------|----------|
| **Member** (회원) | Entity | 높음 | 중간 | 2개 (Loan, Reservation) | 1순위 |
| **Book** (도서) | Entity | 높음 | 중간 | 2개 (Loan, Category) | 1순위 |
| **Loan** (대출) | Entity | 높음 | 높음 | 2개 (Member, Book) | 1순위 |
| **Category** (카테고리) | Entity | 중간 | 낮음 | 1개 (Book) | 2순위 |
| **LoanStatus** | Enum | 중간 | 낮음 | — | 2순위 |
| **MemberStatus** | Enum | 중간 | 낮음 | — | 2순위 |

### 3.2 상세 도메인 객체 정의

#### 3.2.1 Member (회원)

**역할:** 도서관 서비스를 이용하는 일반 회원

**주요 속성:**
- `memberNumber`: 회원번호 (예: M000000001, 자동 생성)
- `name`: 실명
- `email`: 이메일 (로그인 ID 겸용, 고유값)
- `password`: BCrypt 암호화된 비밀번호
- `status`: 회원 상태 (ACTIVE, SUSPENDED, WITHDRAWN)
- `maxLoanCount`: 최대 대출 가능 권수 (기본값: 5)

**주요 행동 (메서드):**
- `canBorrow()`: 대출 가능 여부 확인 (상태 ACTIVE + 현재 대출 수 < maxLoanCount)
- `getCurrentLoanCount()`: 현재 대출 중인 권수 반환
- `suspend()`: 회원 상태를 SUSPENDED로 변경 (관리자 전용)
- `activate()`: 회원 상태를 ACTIVE로 변경 (관리자 전용)
- `withdraw()`: 회원 탈퇴 처리 → WITHDRAWN

**비즈니스 규칙:**
- 이메일은 유일(UNIQUE)해야 하며 로그인 ID로 사용
- SUSPENDED 상태인 회원은 신규 대출 불가
- 연체 중인 회원은 추가 대출 불가
- 회원 탈퇴(WITHDRAWN) 시 대출이력은 보존

#### 3.2.2 Book (도서)

**역할:** 도서관이 보유한 물리적 도서

**주요 속성:**
- `isbn`: ISBN 13자리 (고유값)
- `title`: 도서명
- `author`: 저자
- `publisher`: 출판사
- `category`: 카테고리 (FK → Category)
- `totalCopies`: 총 보유 권수
- `availableCopies`: 현재 대출 가능 권수
- `isActive`: 활성 여부 (소프트 삭제용)

**주요 행동 (메서드):**
- `isAvailable()`: 대출 가능 여부 (`availableCopies > 0`)
- `decreaseAvailable()`: 대출 시 가용 재고 -1
- `increaseAvailable()`: 반납 시 가용 재고 +1
- `deactivate()`: 소프트 삭제 처리

**비즈니스 규칙:**
- ISBN은 유일해야 하며 13자리 숫자
- `availableCopies`는 0 이상, `totalCopies` 이하
- 대출 중인 도서는 삭제 불가 (소프트 삭제만 허용)

#### 3.2.3 Loan (대출)

**역할:** 회원과 도서 간의 대출/반납 트랜잭션

**주요 속성:**
- `member`: 대출 신청 회원 (FK → Member)
- `book`: 대출 도서 (FK → Book)
- `loanDate`: 대출일
- `dueDate`: 반납 예정일 (대출일 + 14일)
- `returnDate`: 실제 반납일 (반납 전 null)
- `status`: 대출 상태 (LoanStatus Enum)

**주요 행동 (메서드):**
- `isOverdue()`: 연체 여부 확인 (`dueDate.isBefore(LocalDate.now()) && status != RETURNED`)
- `getOverdueDays()`: 연체 일수 계산
- `calculateOverdueFee()`: 연체료 계산 (연체 일수 × 100원)
- `approve()`: 상태 REQUESTED → BORROWED
- `returnBook()`: 상태 BORROWED/OVERDUE → RETURNED, returnDate 설정

**비즈니스 규칙:**
- 동일한 회원이 같은 도서를 중복 대출 불가 (BORROWED 상태 존재 시)
- 반납 예정일은 대출 승인일 + 14일로 자동 설정
- 연체료는 하루 100원

#### 3.2.4 Category (카테고리)

**역할:** 도서 분류 체계

**주요 속성:**
- `name`: 카테고리명 (소설, IT, 역사, 과학, 기타)
- `description`: 카테고리 설명

**비즈니스 규칙:**
- 카테고리명은 유일(UNIQUE)
- 도서가 1권 이상 속한 카테고리는 삭제 불가

## 4. 도메인 관계 모델 (ERD 개념)

```
┌──────────────┐        ┌──────────────┐
│   Category   │        │    Member    │
│──────────────│        │──────────────│
│ id (PK)      │        │ id (PK)      │
│ name         │        │ memberNumber │
│ description  │        │ name         │
└──────┬───────┘        │ email        │
       │ 1              │ password     │
       │                │ status       │
       │ N              │ maxLoanCount │
┌──────┴───────┐        └──────┬───────┘
│     Book     │               │ 1
│──────────────│               │
│ id (PK)      │               │ N
│ isbn         │        ┌──────┴───────┐
│ title        │        │     Loan     │
│ author       │        │──────────────│
│ totalCopies  │ 1    N │ id (PK)      │
│ available    ├────────┤ member_id    │
│ isActive     │        │ book_id      │
└──────────────┘        │ loanDate     │
                        │ dueDate      │
                        │ returnDate   │
                        │ status       │
                        └──────────────┘
```

**관계 요약:**
| 관계 | 설명 |
|------|------|
| Category 1:N Book | 하나의 카테고리에 여러 도서 분류 |
| Member 1:N Loan | 한 회원이 여러 번 대출 |
| Book 1:N Loan | 하나의 도서(ISBN)에 여러 대출 이력 |

## 5. Enum 정의

### 5.1 LoanStatus (대출 상태)

| 값 | 설명 | 전이 가능한 상태 |
|----|------|-----------------|
| `REQUESTED` | 대출 신청 완료, 관리자 승인 대기 | → APPROVED, CANCELLED |
| `APPROVED` | 관리자 승인 완료 | → BORROWED |
| `BORROWED` | 대출 중 | → RETURNED, OVERDUE |
| `RETURNED` | 반납 완료 | (종료) |
| `OVERDUE` | 연체 중 (반납 예정일 초과) | → RETURNED |
| `CANCELLED` | 대출 취소 | (종료) |

### 5.2 MemberStatus (회원 상태)

| 값 | 설명 | 대출 가능 여부 |
|----|------|---------------|
| `ACTIVE` | 정상 활성 회원 | 가능 |
| `SUSPENDED` | 정지 상태 (관리자에 의해) | 불가 |
| `WITHDRAWN` | 탈퇴 처리 | 불가 |
