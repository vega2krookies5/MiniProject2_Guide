## 0. 컴포넌트 설계 프로세스 (작성 전 반드시 읽기)

> 컴포넌트 설계는 "화면 → 코드"로 가는 번역 작업입니다.
> 아래 4단계를 순서대로 따라가면 자연스럽게 컴포넌트 설계서가 완성됩니다.

### STEP 1 — 화면 설계서에서 사각형 그리기

UI 화면설계서의 와이어프레임을 보며 **"반복되거나, 독립적으로 동작하는 덩어리"** 마다 사각형을 그립니다.

```
분리 기준 3가지:
① 재사용 가능한가?       → 여러 화면에서 쓰인다면 공통 컴포넌트로
② 독립적으로 변경되는가?  → 검색창이 바뀌어도 테이블은 그대로여야 한다면 분리
③ 너무 길지 않은가?      → 한 컴포넌트가 100줄을 넘으면 분리를 고려
```

**예시 — 도서 목록 화면에서 컴포넌트 추출:**
```
┌──────────────────────────────────────────────────┐
│ [Header 컴포넌트]                                 │  ← 모든 페이지 공통
├──────────────────────────────────────────────────┤
│ ┌────────────────────────────────────────────┐   │
│ │ [BookSearch 컴포넌트]  [CategoryFilter]     │   │  ← 검색/필터 독립
│ └────────────────────────────────────────────┘   │
│ ┌────────────────────────────────────────────┐   │
│ │ [BookDataGrid 컴포넌트]                    │   │  ← 테이블 독립
│ │  (재고 Badge + 대출 Button 포함)            │   │
│ └────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────┘

+ [LoanDialog 컴포넌트] — 대출 버튼 클릭 시 나타나는 모달
```

### STEP 2 — 컴포넌트 트리(계층 구조) 그리기

사각형을 부모-자식 계층으로 배치합니다.
**부모 컴포넌트**가 상태를 갖고, **자식 컴포넌트**는 Props를 받아 표시만 합니다.

```
[작성 규칙]
- 들여쓰기로 부모-자식 관계 표현
- 오른쪽에 해당 컴포넌트가 담당하는 역할 한 줄 메모
- (Props: ...) 로 전달받는 데이터 명시
- (State: ...) 로 자신이 갖는 상태 명시

BookListPage                     ← 페이지 컴포넌트 (상태 소유자)
  ├── BookSearch                  ← 검색창 (Props: onSearch)
  ├── CategoryFilter              ← 카테고리 선택 (Props: value, onChange)
  ├── BookDataGrid                ← 테이블 (Props: rows, loading, onLoan)
  │     └── LoanButton            ← 대출 버튼 (Props: book, onClick)
  └── LoanDialog                  ← 대출 모달 (Props: open, bookId, onConfirm)
```

### STEP 3 — 상태(State) 위치 결정

> **핵심 원칙**: 상태는 그 값을 **필요로 하는 컴포넌트들의 공통 부모**에 둡니다.

```
판단 질문 3가지:

Q1. 이 상태를 여러 컴포넌트가 공유하는가?
    → YES: 공통 부모 컴포넌트 또는 Zustand 전역 스토어
    → NO:  그 컴포넌트 안의 로컬 상태 (useState)

Q2. 새로고침 후에도 유지되어야 하는가?
    → YES: Zustand + localStorage (persist)
    → NO:  일반 useState 또는 Zustand (비영속)

Q3. 서버에서 가져오는 데이터인가?
    → YES: Zustand 스토어의 액션(fetchXxx)으로 관리
    → NO:  UI 상태 (열림/닫힘, 선택 항목 등)는 로컬 useState
```

**상태 위치 결정 예시:**
| 상태 | 위치 결정 | 이유 |
|------|-----------|------|
| `keyword` (검색 입력값) | 로컬 `useState` | BookListPage 안에서만 사용 |
| `isLoanDialogOpen` | 로컬 `useState` | UI 열림/닫힘 → 서버 무관 |
| `books` (도서 목록) | `bookStore` (Zustand) | 서버 데이터, 여러 컴포넌트 공유 가능 |
| `token` (JWT) | `authStore` (Zustand) | 앱 전체에서 인증 확인 필요 |

## 5. API 연동 설계

### 5.1 사용하는 API 목록
| API 엔드포인트 | 메서드 | 용도 | 호출 시점 | 인증 필요 |
|----------------|--------|------|-----------|-----------|
| `/api/books` | `GET` | 도서 목록 조회 | 페이지 진입 / 검색 | 불필요 |
| `/api/books/:id` | `GET` | 도서 상세 조회 | 도서 선택 시 | 불필요 |
| `/api/loans` | `POST` | 대출 신청 | 대출 버튼 클릭 | 필요 |
| `/api/loans/my` | `GET` | 내 대출이력 | /my/loans 진입 | 필요 |
| `/api/loans/:id/return` | `PATCH` | 반납 신청 | 반납 버튼 클릭 | 필요 |

### 5.2 Request / Response 예시
```javascript
// GET /api/books?keyword=클린코드&category=IT&page=0&size=10
// Response 200
{
  "success": true,
  "data": {
    "content": [
      { "id": 1, "title": "클린 코드", "author": "로버트 마틴",
        "category": "IT", "availableCopies": 2, "totalCopies": 3 }
    ],
    "page": { "number": 0, "size": 10, "totalElements": 25, "totalPages": 3 }
  }
}
```

### 5.3 에러 처리 방식
| HTTP 상태 | 의미 | 처리 방식 |
|-----------|------|-----------|
| `400` | 잘못된 요청 (유효성 오류) | 폼 에러 메시지 표시 |
| `401` | 인증 실패 | 로그인 페이지로 이동 |
| `403` | 권한 없음 | "접근 권한이 없습니다" 토스트 |
| `404` | 리소스 없음 | "해당 도서를 찾을 수 없습니다" |
| `500` | 서버 오류 | "서버 오류. 잠시 후 다시 시도해 주세요" |

## 6. 컴포넌트 구현 골격

> 아래 골격의 주석 순서(로컬 상태 → Zustand → useEffect → 핸들러 → 렌더링)를
> 일관되게 유지하면 코드 가독성이 높아집니다.

```jsx
// src/[경로]/[ComponentName].jsx

import { useState, useEffect } from 'react';
import { useNavigate } from 'react-router-dom';
import { useBookStore } from '@/store/bookStore';
import { useAuthStore } from '@/store/authStore';
import { Button } from '@/components/ui/button';
import { Input } from '@/components/ui/input';
import [ChildComponent] from './components/[ChildComponent]';
import { toast } from 'sonner';

export default function [ComponentName]({ [prop1], [prop2] }) {

  // ── 1. 로컬 상태 ─────────────────────────────
  const [keyword, setKeyword] = useState('');
  const [isDialogOpen, setIsDialogOpen] = useState(false);

  // ── 2. Zustand 전역 상태 ─────────────────────
  const { books, loading, fetchBooks } = useBookStore();
  const { token } = useAuthStore();
  const navigate = useNavigate();

  // ── 3. 사이드 이펙트 (데이터 로드) ───────────
  useEffect(() => {
    fetchBooks({ keyword: '', category: '전체', page: 0 });
  }, []);

  // ── 4. 이벤트 핸들러 ─────────────────────────
  const handleSearch = (kw) => {
    setKeyword(kw);
    fetchBooks({ keyword: kw, page: 0 });
  };

  // ── 5. 조건부 렌더링 (로딩/에러/빈 데이터) ───
  // if (loading) return <LoadingSpinner />;
  // if (error)   return <ErrorMessage />;

  // ── 6. 렌더링 ────────────────────────────────
  return (
    <div className="container mx-auto py-6 space-y-4">
      <Input
        placeholder="도서명 또는 저자명 입력..."
        value={keyword}
        onChange={(e) => setKeyword(e.target.value)}
        onKeyDown={(e) => e.key === 'Enter' && handleSearch(keyword)}
      />
      {!loading && books.length === 0 && (
        <p className="text-center text-neutral-500 py-12">
          검색 결과가 없습니다.
        </p>
      )}
      <[ChildComponent]
        rows={books}
        loading={loading}
        onLoan={(book) => {
          if (!token) { toast.error('로그인이 필요합니다'); navigate('/login'); return; }
          setIsDialogOpen(true);
        }}
      />
    </div>
  );
}
```

## 7. 성능 최적화 (해당 시 체크)

| 최적화 기법 | 적용 시기 | 사용법 |
|-------------|-----------|--------|
